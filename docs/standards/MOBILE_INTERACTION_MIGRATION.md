# 移动端棋盘与触控迁移标准

适用对象：带有棋盘、地图、画布或其他可缩放操作面的移动网页，尤其是含固定底部操作栏的实时对局页面。

## 必须保持的规则

1. 默认状态优先服务页面浏览。操作面未缩放时，单指纵向滑动必须滚动整个页面，不能被棋盘或画布拦截。
2. 放大状态优先服务局部浏览。只有缩放后，单指拖动才用于平移棋盘；拖动超过阈值后才阻止点击落子。
3. 首屏优先展示核心操作面。手机进入对局后，完整棋盘应位于可见区域内，且底部不得被固定操作栏遮挡。
4. 固定元素必须让位。固定底栏使用安全区内边距；页面内容保留等于底栏高度加安全区的底部空间。
5. 手机端隐藏冗余信息，而不是缩小棋盘。房间长链接、重复复制按钮和装饰标题可收纳；保留离开、分享、当前轮次和主要落子操作。

## 代码约定

### 手势状态

- 未缩放：`.board-viewport { touch-action: pan-y; }`
- 缩放后：`.board-viewport[data-zoomed="true"] { touch-action: none; }`
- 指针按下时仅记录坐标；不要立刻 `preventDefault()` 或 `setPointerCapture()`。
- 两指缩放时，或已放大且移动距离超过 8px 时，才捕获指针、阻止默认行为并设置“抑制点击”标记。
- 指针结束后清理拖动、缩放和抑制点击状态，避免下一次点按被误吞。

这套边界可避免“为了支持棋盘拖动而让手机页面完全无法滑动”的回归。

### 布局状态

- 页面最小宽度为 320px；基线测试尺寸为 `390 × 844`，同时覆盖 Playwright `Pixel 5` 项目。
- 移动端房间头部使用单行的“离开 + 分享”布局；隐藏 `.room-title`、长分享 URL 和重复复制按钮。
- 棋盘工具栏保持单行，收起说明文字；完整高级操作放入“更多”菜单。
- 固定底栏高度变化时，同步更新 `.room-shell.has-mobile-bar` 的底部内边距。
- 不能依赖固定像素棋盘高度；棋盘仍以容器宽度的正方形显示。

## 自动化验收

迁移后必须在实际双人入局状态下运行，而不是只渲染静态组件：

```bash
npm run check
npm run test:e2e
```

移动端 E2E 至少断言：

- `.board-viewport` 初始 `data-zoomed="false"`；
- 初始计算样式 `touch-action: pan-y`；
- 棋盘底边小于固定栏顶部（390 × 844 基线中不超过 `770px`）；
- 创建、加入、落子、观战聊天与再来一局没有回归；
- 放大棋盘后，`data-zoomed="true"`，可继续执行拖动平移测试。

## 迁移步骤

1. 复制 `Board.tsx` 中的缩放、拖动阈值与 `data-zoomed` 状态约定。
2. 复制 `styles.css` 的触控规则、移动端头部压缩和固定栏避让规则；按新项目类名替换。
3. 将本项目 `e2e/gomoku.spec.ts` 的手机对局用例改为新项目的真实入局流程。
4. 先运行用例确认旧实现失败，再修改样式或手势逻辑，最后执行完整检查。
5. 发布后用公网地址确认健康检查、页面资源版本和手机视口联机流程。

## 当前项目的实现入口

- 触控与缩放：`src/components/Board.tsx`
- 移动布局、固定栏与触控样式：`src/styles.css`
- 手机回归测试：`e2e/gomoku.spec.ts`
